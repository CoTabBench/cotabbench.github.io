📝 Abstract

Real-World Tables Demand a New Benchmark

Recent advancements in Large Language Models (LLMs) and Multimodal Large Language Models (MLLMs) have significantly propelled their capabilities in table-based question answering. However, existing benchmarks predominantly feature well-structured tables in simplified formats, failing to evaluate models on real-world tables in their native LaTeX and HTML form — whose expressive flexibility gives rise to structural irregularity and content heterogeneity.

We introduce CoTabBench, a large-scale, multi-domain benchmark featuring over 2,717 real-world tables with structural irregularity and more than 8,609 question-answer pairs spanning 10 distinct domains. Each table is provided in two forms: the original markup source and the rendered image, preserving the inherently dual-modal nature of LaTeX and HTML tables.

Real-world LaTeX and HTML tables are inherently dual-modal — they exist simultaneously as structured textual code and as rendered two-dimensional visual objects. Processing either form requires both semantic understanding and visual-spatial reasoning. Yet no existing benchmark systematically evaluates LLMs and MLLMs on complex tables in their native markup form.

We further propose BiSC-Quant, a novel complexity assessment framework that quantitatively validates the structural and content-based challenges within CoTabBench. Extensive experiments reveal significant performance degradation for state-of-the-art models, highlighting the critical role of this benchmark in advancing robust, real-world table understanding.

To support model training, we introduce CoTabInstruct, a large-scale training corpus with over 11,096 tables and 32,645 QA pairs, and present CoTabLLM, a 7B model trained on it that outperforms even leading models like GPT-5.4 on our benchmark.

📊 CoTabBench Details

Benchmark Design & Task Categories

CoTabBench is constructed via a dual-source collection pipeline, rigorous multi-stage filtering, and a five-agent QA annotation framework with automated validation.

What Makes CoTabBench Unique

Existing benchmarks rely on clean, regular Wikipedia tables. CoTabBench sources tables directly in the wild — extracting LaTeX tables from the DocGenome dataset (ArXiv papers) and HTML tables via large-scale web crawling across 10 domains. The result is a benchmark that reflects the full structural and semantic complexity of real-world tables.

Each table is provided in two forms: the original markup source (LaTeX or HTML) and a rendered image, preserving the dual-modal nature of real-world tables. Average table length is 952.3 tokens with 14.9 rows and 8.4 columns on average.

Structural Irregularity — 98.3% of tables contain merged cells; avg. 70.6 merged cells per table
Content Heterogeneity — Cells contain formulas, special symbols (†, *), complex numerics ($1.2M, ±0.5), and long-form text
Dual-Modal Inputs — Each table available as LaTeX/HTML markup and as a rendered image for multimodal evaluation
Multi-Agent Annotation — Five specialized agents (Qwen-Plus-Thinking backbone) with dual-path consistency verification
Human Verification — All 8,609 test instances manually validated by 5 annotators; Cohen's κ = 0.82

🗂️ CoTabInstruct

Instruction-Tuning Corpus

A large-scale training corpus built from the same dual-source pipeline as CoTabBench, strictly partitioned so no training table appears in the evaluation set.

// CoTabInstruct — QA pair example { "table_id": "arxiv_cs_table_1847", "source": "latex", "domain": "science", "task": "table_qa", "question": "What is the F1 score of Model A on the hard subset?", "answer": "68.4", "verification": { "dual_path_consistent": true, "validator_passed": true, "human_verified": true } }

CoTabInstruct is generated by the same multi-agent annotation framework as CoTabBench. Tables are sourced via dual-pronged collection: LaTeX tables from the DocGenome dataset (ArXiv scientific literature) and HTML tables via recursive web crawling across 10 domains. All tables pass the same rigorous rule-based and model-based filtering pipeline before QA generation.

QA pairs are generated and verified through a dual-path consistency check — two independent answer paths must agree; discrepancies escalate to Multi-LLM Voting (DeepSeek-R1 + Gemini 2.5 Flash). A 10% random sample (3,265 examples) was manually reviewed by annotators.

1
Dual-source table collection LaTeX tables from DocGenome (ArXiv) + HTML tables from large-scale recursive web crawling across 10 domains
2
Five-agent annotation framework Question Generator, Question Filter, Extractor, Answer Generator, and Validator — all powered by Qwen-Plus-Thinking
3
Multi-dimensional automated validation Data Consistency, Hierarchical Consistency, and Numerical Consistency checks (Python assert execution for math)
4
Strict train/test partition 11,096 training tables and 32,645 QA pairs — no overlap with the 2,717 CoTabBench test tables
🤖 CoTabLLM

Fine-tuned Open-Source Models

A series of models at 7B and 8B scales fine-tuned on CoTabInstruct via SFT+LoRA, serving as strong baselines for complex table understanding.

CoTabLLM is fine-tuned using SFT+LoRA on CoTabInstruct (batch size 16, max sequence length 4096). Two base models are used: Qwen2.5-7B-Instruct (CoTabLLM-7B) and LLaMA3-8B-Instruct (CoTabLLM-8B). Both models are trained only on the CoTabInstruct training split with no exposure to CoTabBench test tables.

Despite their small parameter count, CoTabLLM models surpass considerably larger general-purpose models on CoTabBench. CoTabLLM-7B achieves 57.3 overall — outperforming GPT-5.4 (54.0), Qwen-Plus-Latest (55.2), and DeepSeek-V3 (54.4). This demonstrates that targeted domain adaptation on CoTabInstruct is more effective than raw model scale for complex table understanding. CoTabLLM-7B also achieves the best Hallucination Detection score of 78.6 across all evaluated models.
