📝 Abstract

Benchmarking Collaborative Table Understanding

Tabular data is one of the most prevalent formats for storing and communicating structured information across enterprise, scientific, and everyday domains. While recent large language models (LLMs) have demonstrated impressive capabilities on isolated table understanding tasks, their ability to engage in sustained, multi-turn reasoning over tabular data within collaborative conversational settings remains poorly understood and systematically under-evaluated.

We introduce CoTabBench, the first large-scale benchmark specifically designed to evaluate the collaborative table understanding capabilities of LLMs — where models must interpret, query, and reason about tables across extended multi-turn dialogues that mirror real-world data analysis workflows.

CoTabBench comprises 6,500+ carefully curated instances spanning five core task categories — Table Question Answering, Cell Lookup, Table Summarization, Multi-turn Dialog, and Data Analysis — collected across 12 diverse domains including finance, healthcare, scientific literature, e-commerce, and government statistics. Each instance captures the full context of a collaborative session, including follow-up questions, clarifications, and iterative refinements that occur naturally when humans and AI systems co-analyze structured data.

We evaluate 12 state-of-the-art LLMs on CoTabBench, revealing significant performance gaps between closed-source frontier models and open-source alternatives, particularly in multi-turn consistency and numerical reasoning. To address these gaps, we present CoTabInstruct, an instruction-tuning dataset of 85,000+ conversation turns, and CoTabLLM, a fine-tuned model that achieves competitive performance with proprietary systems while remaining fully open-source.

Our analysis uncovers key challenges in collaborative table understanding: maintaining coherent context across long conversations, grounding responses in specific table cells, and accurately synthesizing multi-step analytical reasoning. CoTabBench establishes a rigorous and reproducible evaluation framework that we hope will drive progress in this practically important yet underexplored area.

📊 CoTabBench Details

Benchmark Design & Task Categories

CoTabBench is constructed through a rigorous multi-stage pipeline combining automated generation, human annotation, and adversarial filtering to ensure quality and difficulty.

Design Principles

Real-world table interactions rarely occur in isolation. Data analysts follow up on initial queries, request different views of the same table, compare across rows, and seek explanations for anomalies. CoTabBench operationalizes this reality by structuring all instances as sessions — multi-turn exchanges where each turn depends on the context established by prior turns.

Tables in CoTabBench range from compact 5×5 matrices to large 100-row financial reports, with a median size of 8 columns and 32 rows. All tables are stored in a unified JSON format compatible with standard data science tooling.

Multi-turn Sessions — Average 4.8 turns per session; longest sessions exceed 12 turns
Cross-Domain Coverage — Finance, healthcare, science, e-commerce, sports, government & more
Adversarial Filtering — Items solvable by simple lookup or keyword matching were removed
Expert Validation — All gold answers verified by domain experts with inter-annotator agreement ≥ 0.87
Composite Evaluation — Exact match, F1, ROUGE-L, BERTScore, and numerical accuracy metrics

🗂️ CoTabInstruct

Instruction-Tuning Dataset

A large-scale dataset of multi-turn conversations grounded in real tables, designed for supervised fine-tuning of language models on collaborative table tasks.

// CoTabInstruct — conversation example { "table_id": "finance_q3_report_042", "domain": "finance", "num_turns": 6, "conversation": [ { "role": "user", "content": "What was the total revenue in Q3 for the APAC region?" }, { "role": "assistant", "content": "The APAC region reported $4.2B in Q3 revenue...", "grounding": { "rows": [3, 4, 5], "cols": ["Q3_revenue"] } }, { "role": "user", "content": "How does that compare to the prior quarter?" } // ... 3 more turns ] }

CoTabInstruct was constructed by combining three complementary sources: (1) expert-authored conversations collected via a dedicated annotation platform, (2) augmented conversations generated using a pipeline of large-scale table-text alignment, and (3) adversarially filtered synthetic data targeting model weaknesses identified through preliminary evaluation on CoTabBench.

Each conversation includes grounding annotations that specify the exact table cells referenced in each response, enabling models to learn precise grounding alongside fluent generation. The dataset is provided with train/validation/test splits aligned to the CoTabBench evaluation protocol.

1
Expert-authored conversations ~25,000 high-quality sessions curated by domain specialists with cell-level grounding annotations
2
Augmented multi-turn data ~45,000 conversations expanded from single-turn table QA pairs using a controlled generation pipeline
3
Adversarial examples ~15,000 targeted hard examples designed to expose multi-turn consistency and numerical reasoning failures
4
Preference pairs for alignment 10,000+ preference-labeled response pairs for DPO-based preference alignment training
🤖 CoTabLLM

Fine-tuned Open-Source Model

A 7B-parameter open-source model fine-tuned on CoTabInstruct, achieving performance competitive with frontier closed-source systems on CoTabBench.

CoTabLLM is initialized from a strong open-source base model and trained using a two-stage recipe: supervised fine-tuning (SFT) on all 85K CoTabInstruct conversations, followed by Direct Preference Optimization (DPO) on the 10K preference pairs. This training approach teaches the model to produce accurate, grounded, and contextually coherent responses across all five CoTabBench task types.

Despite its substantially smaller parameter count compared to GPT-4o and Claude 3.5 Sonnet, CoTabLLM achieves an overall benchmark score of 68.4, placing it third on the CoTabBench leaderboard — outperforming Gemini 1.5 Pro (67.9) and GPT-4 Turbo (65.2). This demonstrates that targeted instruction tuning on high-quality collaborative table data can substantially close the capability gap between open-source and proprietary models for this task family.

Training Pipeline

Stage 1 — Supervised Fine-Tuning (SFT) Full-parameter SFT on 85K CoTabInstruct conversations; 3 epochs on 8×A100 GPUs; lr = 2e-5 with cosine decay
Stage 2 — Direct Preference Optimization (DPO) DPO training on 10K preference pairs; β = 0.1; 1 epoch; focuses on grounding accuracy and factual precision
Table-aware Tokenization Row/column separator tokens injected during tokenization to improve structural awareness of tabular inputs
Context Window Extension Extended context window to 16K tokens via RoPE scaling to handle long multi-turn sessions with large tables
